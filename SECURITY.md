# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in lint-sage, please report it privately. **Do not open a public GitHub issue.**

Email your report to the project maintainers or use the appropriate internal Viitor Cloud security reporting channel.

When reporting, include:

- A description of the vulnerability.
- Steps to reproduce it.
- The potential impact.
- Any suggested fix, if you have one.

We will acknowledge receipt within 3 business days and aim to provide a resolution or mitigation plan within 14 days.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 3.x     | Yes       |
| < 3.0   | No        |

## Scope

This policy covers the `@vcian/lint-sage` CLI package and its published shared config packages (`@vcian/eslint-config-*`, `@vcian/prettier-config`, `@vcian/commitlint-config`).
